# Liquid cyber y2k interface system

## Purpose

This skill defines a full-spectrum interface aesthetic for applications, operating systems, and digital environments inspired by late 1990s to early 2000s experimental consumer technology, but extended into a complete design language for modern platforms.

This is not limited to media players. This applies to:

- full desktop environments
- mobile operating systems
- app ecosystems
- control panels
- dashboards
- system UI layers
- creative tools
- communication interfaces

The goal is to create interfaces that feel like living objects rather than flat screens. The system should feel tactile, expressive, playful, and futuristic in a way that modern UI has largely abandoned.

## Core idea

The interface is not a layer on top of the device.

The interface is the device.

Buttons, panels, windows, and controls should feel like molded components of a physical object, even when purely digital.

## Visual principles

### 1. Interface as object system

Every UI element should feel like a physical module.

Examples:
- windows as rounded capsules
- sliders as embedded rails inside shells
- buttons as glowing nodes
- panels as detachable components
- menus as floating objects rather than dropdown lists

### 2. Biomorphic layout

Avoid rigid grid dominance. Use:

- soft asymmetry
- rounded clustering
- organic grouping of controls
- radial layouts where appropriate
- clustered modules instead of strict rectangles

### 3. Depth and layering

Use:

- inset screens
- raised controls
- layered shells
- glow separation
- soft shadows

Avoid flat layering unless intentionally contrasting.

### 4. Darkness as system canvas

Use dark backgrounds to let modules float and glow.

The system should feel like:

- a control space
- a dashboard in void
- a digital instrument panel

Each module should have a primary accent color identity.

## Interface components

### Windows

- rounded
- sometimes asymmetrical
- may include protrusions or attached controls
- can feel like devices rather than frames

### Buttons

- circular or capsule
- glowing edges
- soft press depth
- high contrast

### Sliders

- embedded tracks
- luminous indicators
- sometimes vertical equalizer style

### Panels

- detachable feel
- modular grouping
- floating or docked with soft magnetism

### Icons

- semi-3D
- glossy
- object-like rather than flat glyphs

## Interaction style

- smooth transitions
- soft easing
- slight elasticity
- glowing feedback
- hover states feel like energy activation

## Typography

- compact
- readable
- digital
- secondary to form

Use:
- condensed sans
- square techno fonts
- monospaced for system readouts

## Composition

### Desktop or app layout

- modules float rather than snap rigidly
- spacing is intentional and breathable
- layering is visible
- each component has identity

### Mobile layout

- clustered controls
- thumb-friendly blob layouts
- radial or grouped UI patterns

## Emotional tone

The system should feel like:

- a future worth using
- something you want to explore
- a tool that invites interaction
- a personal interface identity

## What to avoid

Avoid:

- flat minimal UI
- sterile white layouts
- generic mobile app patterns
- harsh geometric rigidity
- purely utilitarian layouts
- corporate sameness

## Prompt guidance

When generating UI assets or screens:

- specify translucent shell
- specify glowing controls
- specify rounded biomorphic forms
- specify embedded screens
- specify black or dark background
- specify product-like rendering
- specify system interface context
